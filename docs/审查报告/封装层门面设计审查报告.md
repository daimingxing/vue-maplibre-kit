# vue-maplibre-kit 封装层与门面设计审查报告

**审查日期**: 2026-04-25  
**审查范围**: 主入口、业务入口、几何入口、配置入口、插件入口、核心门面、业务示例

---

## 执行摘要

本次审查发现 **3 个严重问题**、**8 个中等问题**、**12 个轻微问题**。整体架构设计合理，门面模式执行到位，但存在类型安全隐患、错误处理不足、文档缺失等问题。

**关键发现**：
- ✅ 门面模式执行良好，内部实现完全隔离
- ✅ 业务层使用体验优秀，认知负担低
- ⚠️ 类型导出不完整，业务层可能遇到类型缺失
- ⚠️ 错误边界处理不足，缺少统一错误提示
- ⚠️ 部分 API 缺少 JSDoc，学习成本高

## 一、严重问题 (P0)

### 1.1 【类型安全】business.ts 缺少关键类型导出

**问题描述**：
`src/business.ts` 作为业务层主入口，缺少多个高频使用的类型导出，导致业务层需要回退到根入口 `vue-maplibre-kit` 查找类型。

**影响范围**：
- 业务层开发体验下降
- 违背"业务层优先从 business 入口接入"的设计原则
- 增加认知负担

**缺失类型清单**：

```typescript
- MapBusinessFeatureContext (业务要素上下文类型) ✓ 已导出
- MapBusinessSelectionItem (业务选中项类型) ✓ 已导出
```

**证据**：
```typescript
// src/business.ts 第 214-225 行
export type { MapBusinessFeatureContext } from './MapLibre/facades/useMapFeatureQuery';
```

### 1.2 【运行时错误】useBusinessMap 缺少地图实例空值保护

**问题描述**：
`useBusinessMap` 在初始化时未对 `mapRef` 进行空值校验，如果业务层在地图未就绪时调用门面方法，会导致运行时错误。

**影响范围**：
- 业务层可能在 `onMounted` 早期调用门面方法时崩溃
- 错误信息不友好，难以定位问题

**证据**：
```typescript
// src/MapLibre/facades/useBusinessMap.ts 第 170-194 行
export function useBusinessMap(options: UseBusinessMapOptions): UseBusinessMapResult {
  const { mapRef, sourceRegistry } = options;
  
  // 直接传递 mapRef，未做空值校验
  const selection = useMapSelection(mapRef);
  const feature = createBusinessMapFeatureGroup(options);
  // ...
}
```

**修复建议**：
```typescript
// 方案 1: 在门面方法内部做空值保护
export function useBusinessMap(options: UseBusinessMapOptions): UseBusinessMapResult {
  const { mapRef, sourceRegistry } = options;
  
  // 添加空值保护
  const safeMapRef = computed(() => {
    const map = toValue(mapRef);
    if (!map) {
      console.warn('[useBusinessMap] 地图实例尚未就绪');
    }
    return map;
  });
  
  const selection = useMapSelection(safeMapRef);
  // ...
}

// 方案 2: 在每个子门面方法内部做空值保护（推荐）
// 让每个方法返回 { success: false, message: '地图未就绪' }
```

**优先级**: P0  
**预计修复时间**: 2 小时

```typescript
export interface MapDxfExportPluginApi {

// 方案 2: 提供 usePluginApi composable（推荐）
export function useMapDxfExportApi(mapRef: MaybeRefOrGetter<MapLibreInitExpose | null>) {
  const api = computed(() => resolveMapDxfExportApi(toValue(mapRef)));
```

**问题描述**：
部分门面方法返回 `boolean`，部分返回 `{ success: boolean, message: string }`，缺少统一的结果类型。

```typescript
export function clear(): MapActionResult {
  if (!hasFeatures.value) {
    return { success: false, message: '当前没有可清理的线草稿' };

  // ...
```

### 2.3 【错误处理】缺少统一错误边界

**问题描述**：
门面方法内部异常未被捕获，直接抛出到业务层，缺少统一的错误处理机制。

**影响范围**：
- 业务层需要自己 try-catch 每个方法
- 错误信息格式不统一

**证据**：
```typescript
// useMapFeatureActions.ts 中可能抛出异常的代码
const lineFeature = MapLineExtensionTool.extendLine({ ... });
// 如果 extendLine 内部抛出异常，会直接传播到业务层
```

**修复建议**：
```typescript
// 添加统一错误处理装饰器
function withErrorBoundary<T extends (...args: any[]) => any>(
  fn: T,
  fallbackMessage: string
): T {
  return ((...args: any[]) => {
    try {
      return fn(...args);
    } catch (error) {
      console.error('[门面错误]', error);
      return {
        success: false,
        message: fallbackMessage,
        error: error instanceof Error ? error.message : String(error),
      };
    }
  }) as T;
}

// 使用
export const previewLine = withErrorBoundary(
  (options: PreviewLineOptions) => { ... },
  '线草稿生成失败'
);
```

**优先级**: P1  
**预计修复时间**: 6 小时

### 2.6 【性能】useBusinessMap 重复创建门面实例

**问题描述**：
每次调用 `useBusinessMap` 都会创建新的门面实例，如果业务层在多个组件中调用，会导致重复计算。

**影响范围**：
- 性能浪费
- 状态不共享

**证据**：
```typescript
// useBusinessMap.ts 第 170-194 行
export function useBusinessMap(options: UseBusinessMapOptions): UseBusinessMapResult {
  // 每次调用都创建新实例
  const selection = useMapSelection(mapRef);
  const feature = createBusinessMapFeatureGroup(options);
  // ...
}
```

**修复建议**：
```typescript
// 方案 1: 提供单例模式
const businessMapCache = new WeakMap<MapLibreInitExpose, UseBusinessMapResult>();

export function useBusinessMap(options: UseBusinessMapOptions): UseBusinessMapResult {
  const map = toValue(options.mapRef);
  if (!map) {
    // 返回空实现
  }
  
  if (businessMapCache.has(map)) {
    return businessMapCache.get(map)!;
  }
  
  const result = createBusinessMapInstance(options);
  businessMapCache.set(map, result);
  return result;
}

// 方案 2: 文档说明"每个页面只调用一次"（推荐）
```

**优先级**: P1  
**预计修复时间**: 4 小时

```typescript
function deepFreeze<T>(obj: T): Readonly<T> {
  Object.freeze(obj);
  Object.getOwnPropertyNames(obj).forEach((prop) => {
    if (obj[prop] !== null && typeof obj[prop] === 'object') {
      deepFreeze(obj[prop]);
    }
  });
  return obj as Readonly<T>;
}
```
