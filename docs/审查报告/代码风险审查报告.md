# 代码风险审查报告

审查范围：`src/` 主封装层为主，`examples/` 仅用于验证公开 API 的实际消费方式，`docs/` 不作为代码审查对象。

验证情况：已执行 `npm test`，结果为 40 个测试文件、185 个测试全部通过。通过测试说明当前已有单元测试链路稳定，但下面问题来自源码契约交叉核对，现有测试未覆盖到这些行为。

## 高风险问题

### 1. 控件预设工厂曾经默认不会显示控件

严重程度：高

处理结论：已决定移除该公开 API。项目未发布，不保留兼容层；业务侧改为显式 `controls` 或全局 `mapControls`。

证据：
- 旧版 `src/MapLibre/facades/businessPreset.ts` 的控件预设在多个组合中只返回 `{}` 形式的控件配置。
- `src/MapLibre/core/mapLibre-init.vue:5-37` 渲染控件时使用 `controls.xxx?.isUse` 判断是否显示。
- `src/MapLibre/core/mapLibre-init.config.ts:67-87` 只合并全局配置和局部配置，没有把空对象补成 `isUse: true`。
- `src/MapLibre/facades/businessPreset.spec.ts` 已改为确认业务预设模块不再提供控件预设工厂。

问题说明：
旧控件预设和全局 `mapControls` 职责重叠，而且 `minimal`、`basic`、`draw`、`full` 的语义不透明。继续修补 `isUse` 只会保留一条不推荐的接入路径。

建议：
文档和示例统一推荐显式 `controls` 或 `defineMapGlobalConfig({ mapControls })`。

## 中低风险问题

### 2. `MglPopup` 初始化后不会响应 `options` 变化

严重程度：中

证据：
- `src/MapLibre/core/mgl-popup.vue:55-72` 只在初始化时用 `props.options` 创建原生 `Popup`。
- `src/MapLibre/core/mgl-popup.vue:98-146` 后续只监听 `visible` 和 `lngLat`，没有监听 `options`。

问题说明：
业务层运行时修改 `closeOnClick`、`closeButton`、`maxWidth`、`className` 等配置时，已经创建的原生 Popup 不会重建或同步。这个问题不会导致崩溃，但会造成“prop 已变化，UI 行为没变化”的隐性状态不一致。

触发场景：
同一个弹窗组件根据业务状态切换 `maxWidth`、关闭按钮或 className。

建议：
明确设计边界：如果 `options` 不支持热更新，在类型注释和文档里写清楚；如果要支持热更新，则监听关键 options 变化，销毁并重建 Popup，同时保持当前 `visible` 和 `lngLat`。

### 3. TerraDraw 控件部分配置只在首次创建时生效

严重程度：中

证据：
- `src/MapLibre/core/useTerradrawControlLifecycle.ts:145-154` 控件实例启停只跟踪 `isLoaded`、`hasMap`、`isUse`。
- `src/MapLibre/core/useTerradrawControlLifecycle.ts:243-254` 控件只在首次启用时创建一次。
- `src/MapLibre/core/useTerradrawControlLifecycle.ts:251` 注释明确写着“后续行为更新由独立 watcher 接管”。

问题说明：
`interactive`、`lineDecoration`、`snapping` 有独立 watcher，但 `position`、底层 `modeOptions` 等构造参数变化不会自动重建控件。业务层如果把绘图模式、按钮集合或位置做成响应式配置，可能会看到配置已变但控件未变。

触发场景：
业务页面根据权限、任务类型或页面模式动态切换 TerraDraw 工具栏模式。

建议：
要么把这些字段标为“仅初始化读取”，要么为构造期字段建立重建依赖。建议先补测试确认哪些字段应该热更新，避免把所有深层配置都纳入 watch 导致频繁重建。

## 已抽查但未发现明显问题的链路

- 插件宿主销毁链路：`useMapPluginHost.ts` 会停止插件状态 watch、调用插件 `destroy()`，并让旧 API 代理失效。
- 普通图层与插件图层事件解绑：`useMapInteractive.ts`、`usePluginLayerInteractive.ts` 都有对应 `map.off()` 清理。
- 吸附、线草稿、交点、面边线、多选、DXF 的公开入口均已通过 `src/plugins.ts` 与 `package.json exports` 暴露。
- 当前自动化测试覆盖了插件宿主、吸附、交互、业务门面、DXF、面边线等核心链路，`npm test` 通过。
