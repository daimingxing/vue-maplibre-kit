## DOC-01：命令式总览漏列 polygonEdge

### 事实依据

源码 `src/MapLibre/facades/useBusinessMap.ts` 的 `UseBusinessMapPlugins` 已包含六个插件：

- `snap`
- `lineDraft`
- `intersection`
- `polygonEdge`
- `multiSelect`
- `dxfExport`

`docs/vue-maplibre-kit-knowledge/12-命令式能力/index.md` 和 `12-命令式能力/05-plugins命令.md` 已写到 `polygonEdge`，但 `12-命令式能力/01-useBusinessMap总览.md` 的 `plugins` 列表只列了五项，漏掉 `plugins.polygonEdge`。

`docs/vue-maplibre-kit-knowledge/15-API参考/01-business-api.md` 的 `useBusinessMap` 分组表也写成：

```txt
snap、lineDraft、intersection、multiSelect、dxfExport
```

同样漏了 `polygonEdge`。

### 影响

这是知识库当前最明确的过期点。业务开发者如果只看总览/API 参考，会以为 `polygonEdge` 没有纳入统一门面，和插件章节、源码门面不一致。

### 建议修复

- 在 `12-命令式能力/01-useBusinessMap总览.md` 的插件短路径列表中补 `plugins.polygonEdge`。
- 在 `15-API参考/01-business-api.md` 的 `plugins` 分组说明中补 `polygonEdge`。

## DOC-02：useBusinessMap 初始化示例类型写法不一致

### 事实依据

`12-命令式能力/01-useBusinessMap总览.md` 当前示例：

```ts
const mapRef = ref<InstanceType<typeof MapLibreInit> | null>(null);
```

而公开业务入口已经导出 `MapLibreInitExpose`，其他文档和示例更多使用：

```ts
const mapRef = shallowRef<MapLibreInitExpose | null>(null);
```

源码 `src/MapLibre/core/mapLibre-init.types.ts` 也明确把 `MapLibreInitExpose` 作为公开实例类型。

### 影响

`InstanceType<typeof MapLibreInit>` 对 Vue SFC 组件公开实例推导更绕，不利于业务开发者理解。既然库已经公开了 `MapLibreInitExpose`，文档应统一使用公开类型。

### 建议修复

把示例改成：

```ts
import { shallowRef } from "vue";
import {
  MapLibreInit,
  createMapBusinessSourceRegistry,
  useBusinessMap,
  type MapLibreInitExpose,
} from "vue-maplibre-kit/business";

const mapRef = shallowRef<MapLibreInitExpose | null>(null);
const sourceRegistry = createMapBusinessSourceRegistry([]);

const businessMap = useBusinessMap({
  mapRef: () => mapRef.value,
  sourceRegistry,
});
```

这与 `NGGI06.vue`、插件章节和当前门面类型更一致。

## DOC-03：示例索引文件名仍停在 NGGI11

### 事实依据

文件路径是：

```txt
docs/vue-maplibre-kit-knowledge/14-示例索引/01-NGGI00到NGGI11.md
```

但文件标题和正文已经写 `NGGI00 到 NGGI12`，并包含 `NGGI12 polygonEdge 面边线`。

### 影响

链接仍可用，不影响阅读正文。但文件名会误导维护者，以为示例索引仍只覆盖到 `NGGI11`。如果后续脚本或人工按文件名查找，也容易漏看 `NGGI12`。

### 建议修复

可选两种方式：

- 重命名为 `01-NGGI00到NGGI12.md`，并同步更新同目录 `index.md`、`02-按功能查示例.md` 的链接。
- 如果暂不想改路径，就在索引页中说明该文件名为历史遗留，内容已覆盖到 `NGGI12`。

由于用户要求优先编辑现有文件，若后续修复建议采用第二种轻量方式，或在确认后做一次完整链接重命名。
