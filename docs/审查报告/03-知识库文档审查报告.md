# 知识库文档审查报告

审查日期：2026-04-30

审查范围：`docs/vue-maplibre-kit-knowledge/**`。判断文档好坏时已对照当前代码、公开入口、package exports 和 `examples/views/NG/GI/**`，不只阅读文档本身。

## 总体判断

知识库目录覆盖面已经比较全面：入门、公开入口、地图与控件、业务数据源、业务图层、样式、交互、属性编辑、插件、几何、全局配置、命令式能力、逃生通道、示例索引和 API 参考都有独立章节。

当前最大问题不是“缺目录”，而是导航型页面和可复制代码有多处与当前代码不一致。这类问题对外部业务开发者影响很大，因为他们通常会先复制首页、公开入口、插件总览、示例索引和 API 参考里的代码。

## 高风险：多个文档仍使用过期的 `useBusinessMap(mapRef)` 写法

状态：本轮修复。知识库可复制代码已统一改为 `useBusinessMap({ mapRef, sourceRegistry })`。

代码事实：

- `src/MapLibre/facades/useBusinessMap.ts:34` 到 `src/MapLibre/facades/useBusinessMap.ts:40`：`useBusinessMap()` 入参是对象，必须包含 `mapRef` 和 `sourceRegistry`。
- `src/MapLibre/facades/useBusinessMap.ts:196` 到 `src/MapLibre/facades/useBusinessMap.ts:233`：实现同样按对象读取 `mapRef`、`sourceRegistry`。
- `examples/views/NG/GI/NGGI06.vue:59` 到 `examples/views/NG/GI/NGGI06.vue:68`：当前示例使用 `useBusinessMap({ mapRef: () => mapRef.value, sourceRegistry: kit.registry })`。

文档问题：

- `docs/vue-maplibre-kit-knowledge/index.md:33` 到 `docs/vue-maplibre-kit-knowledge/index.md:40`：首页 quick start 写成 `const businessMap = useBusinessMap(mapRef)`。
- `docs/vue-maplibre-kit-knowledge/02-公开入口/02-business业务入口.md:57`：同类旧写法。
- `docs/vue-maplibre-kit-knowledge/02-公开入口/03-plugins插件入口.md:49`：同类旧写法。
- `docs/vue-maplibre-kit-knowledge/02-公开入口/06-插件子路径.md:47`：同类旧写法。

影响：

这是可复制代码级错误。外部项目照抄会直接类型不匹配，也会错过 `sourceRegistry` 这个业务门面核心概念。

建议：

全部改为：

```ts
const businessMap = useBusinessMap({
  mapRef: () => mapRef.value,
  sourceRegistry: registry,
});
```

并在首页解释：`sourceRegistry` 是正式业务 source 的统一注册表，查询、编辑和部分插件都会依赖它。
