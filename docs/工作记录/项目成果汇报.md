# vue-maplibre-kit 项目成果汇报

> 面向业务平台建设的 2D WebGIS 前端能力底座

---

## 【第一页】核心定位、技术选型与架构设计

**1. 项目定位与结论**
* **核心定位**：将复杂的 GIS 底层代码，沉淀为**可发布、可复用、可迁移**的标准化 Vue 组件库。
* **业务定性**：它是一个**完全合格且非常适合**作为平台直接开展实际业务（如设备管理、空间标注、巡检运维等）的 2D WebGIS 前端底座。

**2. 核心技术选型**
* **框架底座**：Vue 3 + TypeScript
* **地图引擎**：基于 maplibre-gl 实现基础渲染。
  * **MapLibre GL**：一个开源的 WebGL 驱动的高性能矢量瓦片地图渲染库，提供流畅的交互体验与强大的地图定制能力。
* **绘制引擎**：选用 TerraDraw，负责点线面等复杂图形绘制。
  * **TerraDraw**：一个跨地图框架的几何绘制库，专攻复杂业务场景下的图形绘制、编辑及交互体验。
* **空间计算**：引入 Turf.js，实现距离、面积、质心的精准计算。
  * **Turf.js**：一款强大的浏览器端空间分析库，支持进行各类复杂的几何数学计算与地理分析。

**3. 核心架构亮点（为业务“降本增效”所做的努力）**
* **门面模式 (Facades)**
  * **彻底黑盒化**：将原生 MapLibre 与 TerraDraw 两套引擎的生命周期和事件监听**统一封装暴露**。业务开发只需调一套“傻瓜式”接口，**完全不用操心 GIS 底层实现**。
  * **三层配置架构**：实现“业务层局部 ➔ 全局 ➔ 默认”的属性和样式配置优先级，既开箱即用，又高度可配。
  * **极易迁移**：老项目接入或新项目启动只需依赖稳定公共出口；未来即使底层 GIS 引擎换血，外部业务代码也无需修改。
* **插件系统 (Plugins)**
  * **积木式扩展**：保持核心轻量，将“高级多选”、“智能吸附”等特定功能做成独立插件，按需插拔。
  * **平台演进**：新业务线可自由组合需要的扩展能力，互不干扰，赋予了底座极强的平台演进能力。

## 【第二页】已落地的核心功能矩阵

**1. 基础渲染与高频交互**
* **分层叠加**：支持指定水平图层的选择与叠加渲染。（对应#3；水平图层指定进行图层加载）
* **高度可配 UI**：实现悬浮弹窗（Popup）组件，支持 Vue Slot（插槽）允许业务层自由传入内容。（对应#9：悬浮图元，悬浮出窗口显示内容，悬浮窗口根据鼠标位置出现）
* **高性能动态效果与样式控制**：基于底层 `feature-state` 机制，支持针对特定图元（如通过图元 ID 或分组）进行精细化的动态样式控制。相比于调用 MapLibre 原生的 `setPaintProperty` 触发图层级重绘或使用 `setData` 刷新整份数据，`feature-state` 仅更新局部状态，性能远超原生全量方法。不仅能实现基础的悬浮（Hover）高亮，还能轻松扩展实现颜色交替变换（如**闪烁效果**）、边框高亮等各种复杂的定制化状态展示。
（对应#4：某类型图元，根据图元ID设定不同颜色；
  #5：图元展示效果，颜色交替变换、边框高亮、边框颜色交替变换；
  #6：上层与引擎之间交互，上层告诉指定图元显示什么效果，引擎进行展示）

**2. 深度定制的绘制与空间计算**
* **自定义与数据导出**：支持点线面绘制及样式自定义，一键获取绘制的 GeoJSON 数据。
* **精准测距**：支持点击计算单条线（或指定分段）的长度，以及整个要素的距离与面积。
（对应#8；测量功能，可以测量一段距离多少）
* **空间几何处理**：支持选中线段（如巷道）并输入长度**动态延长**；支持基于线段动态生成（并更新）固定宽度的**矩形要素（线廊）**。（对应#2；画块功能，沿着中线进行画四边形，固定宽度，长度由输入决定）

**3. 突破性的线型渲染 (SVG 纹理)**
> *打破原生限制，支持使用 SVG 图片替换手绘距离段，提供三种渲染模式：*
* **连续纹理 (`line-pattern`)**：让图案像纹理一样连续铺开（重构了离散与原生两种底层实现，可参数控制无缝切换）。
* **节点重复 (`symbol-repeat`)**：沿着线段重复排列特定的 SVG 小图案。
* **拉伸覆盖 (`segment-stretch`)**：将单张图片拉长覆盖指定的一段线。
（对应#7；画一段距离，这个距离使用svg图片形状进行替换）

**4. 强大的智能交互插件**
* **智能吸附 (Snap)**：提供顶点吸附与线吸附双模式（支持绘制时吸附）；**规则与优先级精细可控**（细化到特定图层或单条线），吸附样式完全自定义。
（对应#1；吸附功能，指定图元类型或者线交叉点可吸附）
* **高级多选 (Selection)**：支持单/多选模式切换。多选模式下支持 `Shift+拖拽` 框选、单击追加或反选；可精准控制允许互动的要素/图层；退出策略灵活可配（清空 `clear` 或保留 `retain`）。（对应#11：巷道中线，经过一堆排线，选择排线起始排，结束排，在起始和结束排之间巷道中线经过的排线都选中）
* **全生命周期数据治理**：一套接口统一实现“普通地图图层要素”与“绘制要素”的属性编辑。

## 总结：业务价值与演进潜力

**它的核心价值在于“解耦与复用”**：
1. **降门槛**：把原本需要深厚 GIS 基础的底层操作，封装成了前端日常开发中最熟悉的 Vue 组件和 Hooks API。业务开发可以像用 Element Plus 拼表单一样拼地图应用。
2. **保资产**：基于门面模式（Facades），如果未来由于商业或技术原因需要更换底层地图引擎，只需在底座内部替换，上层业务的调用代码可以基本保持不变，保护了公司的业务代码资产。
3. **稳步迭代**：积木式的插件系统，意味着这个底座不会随着新需求的增加而变成“屎山”。核心永远只做托管，新业务带来的复杂交互全部收敛到独立插件中。

目前，作为一套 2D WebGIS 底座，它在能力完整度、架构合理性和二次开发体验上均已达标，能够稳健地支撑后续设备管理、空间运维等实际业务的开展。
