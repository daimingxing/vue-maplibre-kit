# 声明式与命令式

适合谁读：不确定应该改响应式数据，还是调用地图命令的开发者。

先读哪篇：[03-响应式数据维护](./03-响应式数据维护.md)。

对应示例：[NGGI02](../../../examples/views/NG/GI/NGGI02.vue)、[NGGI03](../../../examples/views/NG/GI/NGGI03.vue)。

## 声明式适合稳定结构

长期存在的业务 source、图层、样式和字段规则，推荐通过响应式数据和模板声明。

```vue
<MapBusinessSourceLayers :source="source" :layers="layers" />
```

这样 Vue 状态、地图渲染和业务表单能保持同一个事实来源。

## 命令式适合临时动作

运行时添加临时分析图层、调试点、高亮、写入 `feature-state`，可以走 `useBusinessMap`。

```ts
const result = businessMap.layers.setFeatureState("asset-source", "asset-a", {
  active: true,
});
```

## 判断规则

| 场景 | 推荐方式 |
| --- | --- |
| 正式业务要素增删改 | 修改响应式 `FeatureCollection` |
| 正式图层结构 | `MapBusinessSourceLayers` 声明 |
| 临时高亮、闪烁 | `feature-state` 或 `businessMap.effect` |
| 调试用临时 source/layer | `businessMap.layers` |
| 属性面板保存字段 | `businessMap.editor` 或 source registry |

命令式动作可能被后续声明式渲染覆盖；如果状态需要长期保存，应回写业务数据。
