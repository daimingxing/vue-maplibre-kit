# 地图实例与生命周期

适合谁读：需要知道何时能调用地图命令、何时只能等待加载的开发者。

先读哪篇：[00-开始使用/02-最小地图](../00-开始使用/02-最小地图.md)。

对应示例：[NGGI01](../../../examples/views/NG/GI/NGGI01.vue)。

## 实例来源

`MapLibreInit` 通过模板 `ref` 暴露 `MapLibreInitExpose`。业务页面通常不直接操作原生地图，而是把这个 ref 交给 `useBusinessMap`。

```ts
import { shallowRef } from "vue";
import { useBusinessMap, type MapLibreInitExpose } from "vue-maplibre-kit/business";

const mapRef = shallowRef<MapLibreInitExpose | null>(null);
const businessMap = useBusinessMap({ mapRef: () => mapRef.value, sourceRegistry: registry });
```

## 生命周期判断

`mapRef.value` 不为空，只说明 Vue 组件实例已经绑定；地图样式和 source/layer 是否可操作，还要看地图宿主状态。

```ts
/**
 * 判断地图是否适合执行运行时 layer/source 动作。
 * @returns 地图已绑定且宿主已加载时返回 true
 */
function canRunLayerAction(): boolean {
  return Boolean(mapRef.value?.rawHandles.mapInstance.isLoaded);
}
```

## 常见调用时机

- `onMounted` 后可以读取 `mapRef`，但仍要处理地图加载中的情况。
- `mapInteractive.onReady` 适合记录交互层已完成初始化。
- source/layer 的长期结构应通过模板插槽声明。
- 临时 source/layer、feature-state、闪烁等动作可在用户交互后调用 `useBusinessMap` 分组。

## 边界

`rawHandles` 可读取原生地图和控件，但它是底层逃生通道。业务页面优先使用 `businessMap.layers`、`businessMap.feature`、`businessMap.editor` 和 `businessMap.plugins`。
