# 响应式数据维护

适合谁读：需要新增、更新、删除正式业务要素的开发者。

先读哪篇：[02-source-layer-feature](./02-source-layer-feature.md)。

对应示例：[NGGI02](../../../examples/views/NG/GI/NGGI02.vue)。

## 推荐方式

正式业务数据由业务页面维护响应式 `FeatureCollection`。地图组件消费这份数据，但不接管业务数据所有权。

```ts
import { ref } from "vue";
import type { MapCommonFeature, MapCommonFeatureCollection } from "vue-maplibre-kit/business";

const sourceData = ref<MapCommonFeatureCollection>({ type: "FeatureCollection", features: [] });

/**
 * 新增业务要素。
 * @param feature 新要素
 */
function addFeature(feature: MapCommonFeature): void {
  sourceData.value = {
    ...sourceData.value,
    features: [...sourceData.value.features, feature],
  };
}
```

## 更新要素

更新时替换目标 feature，避免原地深层修改带来的响应式不确定性。

```ts
/**
 * 更新单个要素属性。
 * @param featureId 业务要素 ID
 * @param patch 属性补丁
 */
function updateFeature(featureId: string, patch: Record<string, unknown>): void {
  sourceData.value = {
    ...sourceData.value,
    features: sourceData.value.features.map((feature) => {
      if (feature.properties?.id !== featureId) {
        return feature;
      }

      return {
        ...feature,
        properties: { ...feature.properties, ...patch },
      };
    }),
  };
}
```

## 删除要素

```ts
/**
 * 删除单个业务要素。
 * @param featureId 业务要素 ID
 */
function removeFeature(featureId: string): void {
  sourceData.value = {
    ...sourceData.value,
    features: sourceData.value.features.filter((feature) => feature.properties?.id !== featureId),
  };
}
```

如果已经创建了业务 source，也可以使用 `source.replaceFeatures()` 或 registry 的写回方法统一处理。
