# 要素新增更新删除

适合谁读：需要维护正式业务要素集合的开发者。

先读哪篇：[03-MapBusinessSourceLayers](./03-MapBusinessSourceLayers.md)。

对应示例：[NGGI02](../../../examples/views/NG/GI/NGGI02.vue)。

## 新增

```ts
/**
 * 新增一个业务要素。
 * @param feature 新要素
 */
function addFeature(feature: MapCommonFeature): void {
  sourceData.value = {
    ...sourceData.value,
    features: [...sourceData.value.features, feature],
  };
}
```

## 更新

属性编辑推荐走 registry 或 `businessMap.editor`，它们会遵守 `propertyPolicy`。

```ts
const result = registry.saveProperties("asset-source", "asset-a", {
  name: "新名称",
});
```

如果是批量替换、排序、删除，可以直接替换 features 数组。

## 删除

```ts
/**
 * 删除指定业务 ID 的要素。
 * @param featureId 业务要素 ID
 */
function removeFeature(featureId: string): void {
  source.replaceFeatures(
    sourceData.value.features.filter((feature) => feature.properties?.id !== featureId)
  );
}
```

## 边界

正式业务数据不要直接用原生 `removeLayer` 或 `removeSource` 来表达删除。地图 source/layer 是渲染结构，业务要素删除应回到业务数据集合。
