# 交点计算

交点工具用于从业务 source 中提取线要素，计算线线交点，并按需要生成正式点要素。

## 主要导出

| 导出 | 用途 |
| --- | --- |
| `buildIntersectionCandidates(sources)` | 从多个 GeoJSON source 中提取可求交的 `LineString` 候选线 |
| `collectLineIntersections(options)` | 按 `all` 或 `selected` 范围收集交点 |
| `buildIntersectionPointFeature(intersection, extraProperties)` | 把交点领域对象转换为点要素 |
| `buildMaterializedIntersectionFeature(intersection, extraProperties)` | 生成适合持久保留的正式交点点要素 |

## 基本用法

```ts
import {
  buildIntersectionCandidates,
  collectLineIntersections,
  buildMaterializedIntersectionFeature,
} from 'vue-maplibre-kit/geometry';

/**
 * 计算多个业务线 source 的全部交点。
 * @param sources 参与求交的业务 source 描述
 * @returns 正式交点点要素列表
 */
export function buildIntersectionPoints(sources) {
  const candidates = buildIntersectionCandidates(sources);
  const intersections = collectLineIntersections({
    scope: 'all',
    candidates,
    includeEndpoint: true,
  });

  return intersections.map((intersection) => {
    return buildMaterializedIntersectionFeature(intersection);
  });
}
```

## `selected` 范围

`scope: 'selected'` 表示只把当前选中线作为左侧候选，与候选集合内其他线求交。

```ts
import { collectLineIntersections } from 'vue-maplibre-kit/geometry';

/**
 * 只计算当前选中线与其他线的交点。
 * @param candidates 当前候选线集合
 * @param selectedRef 当前选中线来源引用
 * @returns 交点领域对象列表
 */
export function collectSelectedIntersections(candidates, selectedRef) {
  return collectLineIntersections({
    scope: 'selected',
    selectedRef,
    candidates,
    includeEndpoint: false,
  });
}
```

## 候选来源结构

`buildIntersectionCandidates()` 接收的单个来源包含：

- `sourceId`：业务 source ID。
- `layerId`：业务图层 ID，用于后续属性治理或回源定位。
- `data`：GeoJSON 数据，通常来自 `source.sourceProps.data`。

只有 `LineString` 且能解析出业务 ID 的要素会进入候选集合。

## 边界

- `IntersectionScope` 只有 `all` 和 `selected`。
- `includeEndpoint` 控制是否保留端点相交，默认保留。
- `coordDigits` 默认保留 6 位小数，用于生成稳定交点 ID。
- 重合线段和无法收敛为单点的情况会被忽略。
- 若 `ignoreSelf` 为默认 `true`，同一条线不会与自己求交。
