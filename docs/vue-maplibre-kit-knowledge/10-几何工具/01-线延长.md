# 线延长

线延长能力由 `MapLineExtensionTool` 提供。它适合在业务层根据点击线段生成一条临时延长线，再交给线草稿插件或业务 source 展示。

## 常用方法

| 方法 | 用途 |
| --- | --- |
| `resolveLineInteractionSnapshot(options)` | 用点击上下文解析完整线要素和命中线段 |
| `resolveNearestSegmentSelection(coordinates, lngLat)` | 只根据坐标和点击点计算最近线段 |
| `extendSelectedLineSegment(lineFeature, segmentIndex, extendLengthMeters, origin)` | 沿命中线段方向生成临时延长线 |
| `isTemporaryExtensionFeature(feature)` | 判断要素是否为延长线草稿 |
| `getLineLengthInMeters(coordinates)` | 计算折线总长度 |
| `normalizeLineCoordinates(coordinates)` | 过滤连续重复坐标，避免零长度线段参与计算 |

## 典型流程

```ts
import {
  MapLineExtensionTool,
  createMapSourceFeatureRef,
} from 'vue-maplibre-kit/geometry';

/**
 * 根据点击上下文创建线延长草稿。
 * @param context 图层交互上下文
 * @param extendLengthMeters 延长长度，单位米
 * @returns 延长线草稿；无法生成时返回 null
 */
export function createLineDraft(context, extendLengthMeters: number) {
  const featureRef = createMapSourceFeatureRef(
    context.sourceId,
    context.featureId,
    context.layerId
  );
  const snapshot = MapLineExtensionTool.resolveLineInteractionSnapshot({
    feature: context.feature,
    featureRef,
    lngLat: context.lngLat,
    resolveLatestFeature: context.resolveLatestFeature,
  });

  if (!snapshot?.segmentSelection) {
    return null;
  }

  return MapLineExtensionTool.extendSelectedLineSegment(
    snapshot.lineFeature,
    snapshot.segmentSelection.index,
    extendLengthMeters,
    featureRef
  );
}
```

## 边界

- `extendLengthMeters` 必须大于 `0`，业务层应在调用前校验。
- 命中线段索引必须落在当前折线坐标范围内，否则返回 `null`。
- 输入线要素必须能解析出业务 ID，优先来自 `properties.id`，其次来自 GeoJSON 顶层 `id`。
- 生成的临时线会写入 `generatedKind`、`generatedFromLineId`、`generatedFromSegmentIndex` 和来源引用属性。

## 与 `useBusinessMap` 的关系

业务页面通常不需要直接调用本工具。若已经注册线草稿插件，推荐从 `businessMap.feature.previewLine()` 或 `businessMap.feature.previewSelectedLine()` 进入，它们内部会调用同类几何逻辑并处理插件缺失、选中态和错误消息。
