# 来源引用

来源引用用于在地图交互、属性编辑、预览要素和插件动作之间稳定描述一个正式业务要素。标准结构是 `MapSourceFeatureRef`。

```ts
interface MapSourceFeatureRef {
  sourceId: string | null;
  featureId: string | number | null;
  layerId?: string | null;
}
```

## 常用导出

| 导出 | 用途 |
| --- | --- |
| `createMapSourceFeatureRef(sourceId, featureId, layerId)` | 把来源信息归一化为标准引用 |
| `buildMapSourceFeatureRefKey(featureRef)` | 生成稳定字符串键 |
| `buildManagedPreviewOriginProperties(featureRef)` | 把正式来源写入草稿或预览要素属性 |
| `extractManagedPreviewOriginFromProperties(properties)` | 从草稿或预览要素属性还原正式来源 |
| `MANAGED_PREVIEW_ORIGIN_*` | 托管来源属性名常量 |

## 创建引用

```ts
import { createMapSourceFeatureRef } from 'vue-maplibre-kit/geometry';

/**
 * 从图层交互上下文创建来源引用。
 * @param context 普通图层交互上下文
 * @returns 标准来源引用；信息不足时返回 null
 */
export function toFeatureRef(context) {
  return createMapSourceFeatureRef(context.sourceId, context.featureId, context.layerId);
}
```

`sourceId` 为空或 `featureId` 为空时会返回 `null`。`featureId` 的数字 `0` 是合法值，不会被当成空值。

## 草稿来源追踪

线延长、线廊、交点预览这类托管要素经常需要回到正式 source。可以把正式来源写入预览属性：

```ts
import {
  buildManagedPreviewOriginProperties,
  extractManagedPreviewOriginFromProperties,
} from 'vue-maplibre-kit/geometry';

/**
 * 为预览要素补充来源属性。
 * @param featureRef 正式来源引用
 * @returns 可合并进 properties 的属性片段
 */
export function buildPreviewProps(featureRef) {
  return buildManagedPreviewOriginProperties(featureRef);
}

/**
 * 从预览要素属性恢复正式来源。
 * @param properties 预览要素属性
 * @returns 正式来源引用；来源不完整时返回 null
 */
export function getOriginRef(properties) {
  return extractManagedPreviewOriginFromProperties(properties);
}
```

## 与业务门面的关系

业务页面优先使用 `businessMap.sources.createFeatureRef()` 或 `businessMap.feature.getFeatureRef()`。只有在独立工具函数、测试数据或非 Vue 场景里，才需要直接从 `vue-maplibre-kit/geometry` 引入来源引用工具。
