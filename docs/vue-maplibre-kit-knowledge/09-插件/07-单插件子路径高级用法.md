# 单插件子路径高级用法

常规业务页优先使用 `vue-maplibre-kit/plugins` 的 `createBusinessPlugins` 注册插件，并通过 `useBusinessMap().plugins.*` 读取能力。单插件子路径适合高级场景：读取常量、使用高级类型、按需注册单个插件、组合底层插件描述。

## 推荐优先级

1. 业务常规接入：`createBusinessPlugins`。
2. 业务统一读取：`useBusinessMap().plugins.*`。
3. 高级类型、常量、单插件工厂：单插件子路径。
4. 内部实现：不要直接引用 `src/MapLibre/**`。

## 子路径能力

`src/entries/plugins.ts` 已经从聚合入口导出了六个插件的大部分能力；真实包消费时，建议先从 `vue-maplibre-kit/plugins` 获取类型和工厂。只有需要更明确的子路径边界时，再使用单插件子路径。

| 插件 | 聚合入口能力 | 子路径用途 |
| --- | --- | --- |
| snap | `createMapFeatureSnapPlugin`、`MapFeatureSnapRule`、预览常量 | 只注册吸附、直接读取吸附常量或高级类型 |
| lineDraft | `createLineDraftPreviewPlugin`、草稿 source / layer / generatedKind 常量 | 只注册线草稿、识别草稿类型、图层或 source |
| intersection | `createIntersectionPreviewPlugin`、交点 source / layer / generatedKind 常量、候选线工具 | 自行构建候选线、识别预览和正式交点 |
| polygonEdge | `createPolygonEdgePreviewPlugin`、面边线 generatedKind / source / layer 常量和类型 | 只注册面边线、识别边线类型、图层或 source |
| multiSelect | `createMapFeatureMultiSelectPlugin`、多选类型 | 只注册多选插件、读取插件类型常量 |
| dxfExport | `createMapDxfExportPlugin`、DXF 默认常量和类型 | 自定义导出控件、读取 DXF 默认值和 resolver 类型 |

## 按需注册单插件

如果页面不需要业务预设，也可以按需创建单个插件描述。但读取仍推荐走 `businessMap.plugins.*`，这样业务代码不会分裂成多套读取方式。

```ts
import { createMapFeatureMultiSelectPlugin } from "vue-maplibre-kit/plugins";

const plugins = [
  createMapFeatureMultiSelectPlugin({
    enabled: true,
    targetLayerIds: [lineLayerId],
  }),
];

const multiSelect = businessMap.plugins.multiSelect;
```

## 读取常量

常量适合用于图层识别、调试面板、样式覆盖或测试断言。

```ts
import {
  INTERSECTION_MATERIALIZED_KIND,
  INTERSECTION_PREVIEW_LAYER_ID,
  INTERSECTION_PREVIEW_KIND,
  LINE_DRAFT_PREVIEW_EXTENSION_KIND,
  LINE_DRAFT_PREVIEW_CORRIDOR_KIND,
  LINE_DRAFT_PREVIEW_SOURCE_ID,
  MAP_DXF_EXPORT_PLUGIN_TYPE,
  MAP_FEATURE_SNAP_PREVIEW_POINT_LAYER_ID,
  POLYGON_EDGE_PREVIEW_KIND,
} from "vue-maplibre-kit/plugins";
```

生成要素的 `generatedKind` 也会作为公开常量导出，当前完整清单如下：

| 插件 | generatedKind | 公开常量 |
| --- | --- | --- |
| lineDraft | `line-extension-draft` | `LINE_DRAFT_PREVIEW_EXTENSION_KIND` |
| lineDraft | `line-corridor-draft` | `LINE_DRAFT_PREVIEW_CORRIDOR_KIND` |
| intersection | `intersection-preview` | `INTERSECTION_PREVIEW_KIND` |
| intersection | `intersection-materialized` | `INTERSECTION_MATERIALIZED_KIND` |
| polygonEdge | `polygon-edge-preview` | `POLYGON_EDGE_PREVIEW_KIND` |

snap、multiSelect、dxfExport 不创建业务生成要素，因此没有 `generatedKind` 常量。

## 高级类型

当业务页需要给配置对象添加显式类型时，可以从公开插件入口读取类型。

```ts
import type {
  IntersectionPreviewOptions,
  LineDraftPreviewOptions,
  MapDxfExportTaskOptions,
  MapFeatureMultiSelectOptions,
  MapFeatureSnapOptions,
  PolygonEdgePreviewOptions,
} from "vue-maplibre-kit/plugins";
```

## 仍然统一读取

即使通过单插件工厂注册，业务读取也保持统一：

```ts
const snap = businessMap.plugins.snap;
const draft = businessMap.plugins.lineDraft;
const intersection = businessMap.plugins.intersection;
const polygonEdge = businessMap.plugins.polygonEdge;
const multiSelect = businessMap.plugins.multiSelect;
const dxfExport = businessMap.plugins.dxfExport;
```

## 示例引用

- `NGGI06.vue` 到 `NGGI12.vue` 都使用聚合入口 `vue-maplibre-kit/plugins`。
- 当前示例没有直接从单插件子路径导入；这符合“业务层优先聚合入口，子路径留给高级定制”的设计。

## 风险提示

- 单插件子路径不应成为业务默认写法，否则会削弱 `createBusinessPlugins` 和 `useBusinessMap().plugins.*` 的一致性。
- 子路径适合公开常量和高级类型，不代表可以绕过门面直接引用内部 `src/MapLibre/**`。
- 如果单插件按需注册后仍读取不到能力，优先检查 `MapLibreInit` 是否收到 `plugins`，以及地图实例是否已经初始化完成。
