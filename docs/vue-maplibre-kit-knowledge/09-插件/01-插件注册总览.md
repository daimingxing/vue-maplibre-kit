# 插件注册总览

业务插件的推荐路径是：注册时使用 `createBusinessPlugins`，读取时使用 `useBusinessMap().plugins.*`。这条路径把插件声明、地图注册、状态读取和命令式动作收口到公开门面，符合组件库对外消费的边界。

## 推荐导入

```ts
import {
  MapBusinessSourceLayers,
  MapLibreInit,
  useBusinessMap,
  type MapLibreInitExpose,
} from "vue-maplibre-kit/business";
import { createBusinessPlugins } from "vue-maplibre-kit/plugins";
```

不要在业务页中直接引用 `src/MapLibre/**`。单插件子路径只保留给深度定制、常量和高级类型场景。

## 注册方式

`createBusinessPlugins` 接收 `BusinessPluginsOptions`，返回可直接传给 `MapLibreInit` 的插件描述数组。

```ts
const plugins = createBusinessPlugins({
  snap: { layerIds: [lineLayerId] },
  lineDraft: true,
  intersection: {
    enabled: true,
    visible: true,
    targetSourceIds: [source.sourceId],
    targetLayerIds: [lineLayerId],
    sourceRegistry: registry,
  },
  multiSelect: {
    enabled: true,
    targetLayerIds: [pointLayerId, lineLayerId],
  },
  dxfExport: {
    enabled: true,
    sourceRegistry: registry,
    control: { enabled: false },
  },
});
```

```vue
<MapLibreInit ref="mapRef" :plugins="plugins" />
```

## 读取方式

`useBusinessMap` 会把五个插件统一挂到 `plugins` 分组。

```ts
const businessMap = useBusinessMap({
  mapRef: () => mapRef.value,
  sourceRegistry: registry,
});

const snap = businessMap.plugins.snap;
const draft = businessMap.plugins.lineDraft;
const intersection = businessMap.plugins.intersection;
const multiSelect = businessMap.plugins.multiSelect;
const dxfExport = businessMap.plugins.dxfExport;
```

## 配置键与读取路径

| 配置键 | 读取路径 | 主要配置 | 主要动作 |
| --- | --- | --- | --- |
| `snap` | `plugins.snap` | `layerIds`、`ordinaryLayers.rules`、`preview`、`terradraw` | `clearPreview`、`resolveMapEvent`、`resolveTerradrawSnapOptions` |
| `lineDraft` | `plugins.lineDraft` | `enabled`、`styleOverrides`、草稿交互回调 | `previewLine`、`replacePreviewRegion`、`saveProperties`、`removeProperties`、`clear` |
| `intersection` | `plugins.intersection` | `targetSourceIds`、`targetLayerIds`、`sourceRegistry`、`includeEndpoint`、`coordDigits` | `refresh`、`materialize`、`removeMaterialized`、`updateMaterializedProperties`、`show`、`hide` |
| `multiSelect` | `plugins.multiSelect` | `enabled`、`targetLayerIds`、`deactivateBehavior`、`canSelect`、`position` | `activate`、`deactivate`、`toggle`、`clear`、`getSelectedFeatures` |
| `dxfExport` | `plugins.dxfExport` | `sourceRegistry`、`defaults`、`control` | `exportDxf`、`downloadDxf`、`getResolvedOptions` |

## true 简写

`businessPreset.ts` 中提供了简写能力：

- `snap: true` 会启用基础吸附能力；普通业务图层吸附还需要传 `layerIds` 或 `ordinaryLayers`。
- `lineDraft: true` 会等价为 `{ enabled: true }`。
- `multiSelect: true` 会等价为 `{ enabled: true }`。

`intersection` 和 `dxfExport` 需要必要业务上下文，通常直接传对象配置。

## 风险提示

- 插件动作多数依赖地图组件实例初始化完成；在插件未注册或地图未就绪时，门面会返回 `false`、`null` 或空集合。
- 业务页应先注册插件，再读取对应 `businessMap.plugins.*` 动作；只读取门面但没有传入 `plugins` 不会自动安装插件。
- 示例里的 `kit.registry`、`kit.source`、图层 ID 来自业务模拟层，真实项目需要替换成自己的业务 source 和图层。
