# 单插件工厂注册

常规业务页优先使用 `createBusinessPlugins()`，它会承接业务预设、简便写法、顶层 `sourceRegistry` 复用和统一默认值。单插件工厂 `createXXXXPlugin(options)` 仍然保留，定位是高级入口：当业务需要绕过预设层、精确传入底层插件配置、只注册一个插件，或在封装自己的业务预设时使用。

单插件工厂不建议去除。它们是公开插件描述对象的直接创建入口，也是 `createBusinessPlugins()` 的底层能力来源。去除后会让高级用户只能通过业务预设层绕路，反而削弱组件库作为 npm 库的可组合性。

## 导入方式

```ts
import {
  createMapFeatureSnapPlugin,
  createLineDraftPreviewPlugin,
  createIntersectionPreviewPlugin,
  createPolygonEdgePreviewPlugin,
  createMapFeatureMultiSelectPlugin,
  createMapDxfExportPlugin,
} from "vue-maplibre-kit/plugins";
```

这些工厂也可以从单插件子路径导入。普通业务项目优先使用聚合入口 `vue-maplibre-kit/plugins`，子路径更适合只取某个插件的常量、类型或工厂。

## 工厂速查

| 工厂 | 参数类型 | 典型用途 |
| --- | --- | --- |
| `createMapFeatureSnapPlugin(mapFeatureSnapOptions)` | `MapFeatureSnapOptions` | 只注册吸附插件，直接传完整 `businessLayers.rules`、预览、TerraDraw 和内置目标配置 |
| `createLineDraftPreviewPlugin(lineDraftPreviewOptions)` | `LineDraftPreviewOptions` | 只注册线草稿插件，精确控制草稿样式、交互回调和属性写回 |
| `createIntersectionPreviewPlugin(intersectionPreviewOptions)` | `IntersectionPreviewOptions` | 只注册交点插件，直接配置求交范围、候选线来源和正式交点行为 |
| `createPolygonEdgePreviewPlugin(polygonEdgePreviewOptions)` | `PolygonEdgePreviewOptions` | 只注册面边线预览插件，控制边线生成、高亮、选择和样式规则 |
| `createMapFeatureMultiSelectPlugin(mapFeatureMultiSelectOptions)` | `MapFeatureMultiSelectOptions` | 只注册多选插件，控制目标图层、选择行为和控件位置 |
| `createMapDxfExportPlugin(mapDxfExportOptions)` | `MapDxfExportOptions` | 只注册 DXF 导出插件，直接传 source 注册表、导出默认值和控件配置 |

## 使用示例

```ts
import {
  createMapFeatureSnapPlugin,
  createMapFeatureMultiSelectPlugin,
} from "vue-maplibre-kit/plugins";

const plugins = [
  createMapFeatureSnapPlugin({
    enabled: true,
    control: {
      enabled: true,
      position: "top-left",
      panel: { enabled: true },
    },
    businessLayers: {
      enabled: true,
      rules: [
        {
          id: "main-line-snap",
          label: "主正式线",
          layerIds: [lineLayerId],
          snapTo: ["vertex", "segment"],
          priority: 30,
        },
      ],
    },
  }),
  createMapFeatureMultiSelectPlugin({
    enabled: true,
    targetLayerIds: [lineLayerId],
  }),
];
```

模板中仍然把 `plugins` 传给地图容器：

```vue
<MapLibreInit ref="mapRef" :plugins="plugins" />
```

## 和 createBusinessPlugins 的区别

`createBusinessPlugins()` 是业务推荐写法，适合真实项目页面直接接入：

```ts
const plugins = createBusinessPlugins({
  sourceRegistry: kit.registry,
  snap: {
    businessLayers: {
      主正式线: lineLayerId,
    },
  },
  intersection: {
    targetLayerIds: [lineLayerId],
  },
  dxfExport: true,
});
```

单插件工厂使用底层完整配置，不处理业务预设层的简写。例如 `createMapFeatureSnapPlugin()` 不支持 `snap.layerIds` 和命名 `businessLayers` 简写，必须传完整的 `businessLayers: { enabled, rules }`。

| 能力 | `createBusinessPlugins()` | `createXXXXPlugin()` |
| --- | --- | --- |
| 多插件统一注册 | 支持 | 需要业务自己组装数组 |
| `sourceRegistry` 顶层复用 | 支持 | 需要逐插件传入 |
| `snap.layerIds` 超简写 | 支持 | 不支持 |
| `snap.businessLayers` 命名简写 | 支持 | 不支持 |
| `lineDraft: true` 等布尔简写 | 支持 | 不支持 |
| 直接传底层完整 options | 大部分支持 | 支持 |
| 适合封装自定义业务预设 | 可以作为上层入口 | 适合作为底层拼装入口 |

## 读取方式保持一致

无论通过 `createBusinessPlugins()` 还是 `createXXXXPlugin()` 注册，业务读取都推荐走 `useBusinessMap().plugins.*`：

```ts
const snap = businessMap.plugins.snap;
const multiSelect = businessMap.plugins.multiSelect;
```

这样业务代码不会因为注册方式不同而分裂成多套读取路径。

## 风险提示

- 单插件工厂不会帮你补业务预设默认值；传入配置要符合对应插件的完整 options 类型。
- `createIntersectionPreviewPlugin()` 自动求交模式仍需要明确 `targetSourceIds` 或 `targetLayerIds`，并提供可读取业务数据的来源。
- `createMapDxfExportPlugin()` 不会自动读取 `createBusinessPlugins({ sourceRegistry })` 的顶层上下文，需要自己传 `sourceRegistry`。
- 常规业务页面不应为了“更底层”而优先使用单插件工厂；只有需要精确控制、按需组合或自建预设时再使用。
