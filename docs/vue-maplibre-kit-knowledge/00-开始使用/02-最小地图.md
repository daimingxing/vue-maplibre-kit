# 最小地图

适合谁读：已经安装依赖，希望先把地图挂起来的开发者。

先读哪篇：[01-安装与样式](./01-安装与样式.md)。

对应示例：[NGGI01](../../../examples/views/NG/GI/NGGI01.vue)。

## 最小组件

```vue
<template>
  <MapLibreInit ref="mapRef" :map-options="mapOptions" :controls="controls" />
</template>

<script setup lang="ts">
import { shallowRef } from "vue";
import {
  MapLibreInit,
  createMapControlsPreset,
  type MapLibreInitExpose,
  type MapOptions,
} from "vue-maplibre-kit/business";

const mapRef = shallowRef<MapLibreInitExpose | null>(null);

const mapOptions: Partial<MapOptions & { mapStyle: object }> = {
  mapStyle: {
    version: 8,
    sources: {},
    layers: [{ id: "bg", type: "background", paint: { "background-color": "#eef2f3" } }],
  },
  center: [113.9, 22.5],
  zoom: 10,
};

const controls = createMapControlsPreset("minimal", {
  MglNavigationControl: { isUse: true, position: "top-left" },
  MglScaleControl: { isUse: true, position: "bottom-left" },
});
</script>
```

## `mapRef` 是什么

`mapRef` 指向 `MapLibreInit` 暴露的公开实例。后续的命令式能力，例如运行时添加图层、写入 `feature-state`、读取插件 API，都从这个公开实例或 `useBusinessMap` 开始。

## 加载状态

示例页用 `mapRef.value?.rawHandles.mapInstance.isLoaded` 展示加载态。真实业务中，建议在地图已挂载、样式已加载、source 和 layer 已声明后，再调用运行时图层动作。

```ts
/**
 * 判断当前地图是否完成基础加载。
 * @returns 地图宿主加载完成时返回 true
 */
function isMapReady(): boolean {
  return Boolean(mapRef.value?.rawHandles.mapInstance.isLoaded);
}
```

`rawHandles` 是底层逃生句柄，普通业务逻辑优先使用 `useBusinessMap`，只有需要读取宿主加载态或原生实例时再下探。

## 什么时候调用命令式能力

- 模板已经渲染出 `MapLibreInit`。
- `mapRef.value` 不为空。
- 对 source/layer 的操作最好等地图样式已加载。
- 对正式业务数据的增删改，优先修改业务侧响应式数据，而不是直接操作原生地图。
