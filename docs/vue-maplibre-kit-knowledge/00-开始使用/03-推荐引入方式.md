# 推荐引入方式

适合谁读：准备在真实业务页面组织 import 的开发者。

先读哪篇：[02-最小地图](./02-最小地图.md)。

对应示例：[NGGI02](../../../examples/views/NG/GI/NGGI02.vue)、[NGGI05](../../../examples/views/NG/GI/NGGI05.vue)。

## 业务页面默认入口

```ts
import {
  MapLibreInit,
  MapBusinessSourceLayers,
  useBusinessMap,
  createMapBusinessSource,
  createMapBusinessSourceRegistry,
} from "vue-maplibre-kit/business";
```

`vue-maplibre-kit/business` 面向业务项目消费，集中暴露地图根组件、业务 source、业务图层、交互上下文、属性编辑和高层聚合门面。

## 插件注册入口

```ts
import { createBusinessPlugins } from "vue-maplibre-kit/plugins";
```

`createBusinessPlugins` 的推荐归属是 `vue-maplibre-kit/plugins`。业务入口中保留了兼容导出，但新页面建议从插件入口引入。

## 全局配置和几何工具

```ts
import { setMapGlobalConfig } from "vue-maplibre-kit/config";
import { MapLineMeasureTool } from "vue-maplibre-kit/geometry";
```

全局默认值只从 `config` 入口接入，几何计算只从 `geometry` 入口接入。这样外部项目升级组件库时，依赖边界更清楚。

## 不推荐路径

不要在业务项目里引用组件库内部目录。文档和示例默认只展示包名路径，除非进入高级章节说明底层逃生通道。
