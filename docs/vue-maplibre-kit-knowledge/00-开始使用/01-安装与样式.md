# 安装与样式

适合谁读：准备把组件库安装到外部 Vue 3 项目的开发者。

先读哪篇：[开始使用索引](./index.md)。

对应示例：当前没有专门安装示例；接入后的最小运行方式见 [NGGI01](../../../examples/views/NG/GI/NGGI01.vue)。

## 安装依赖

组件库声明了 `peerDependencies`，意思是这些基础库由外部项目安装并统一管理版本。业务项目至少需要安装 Vue、MapLibre GL、vue-maplibre-gl、TerraDraw 相关依赖，以及本组件库。

```powershell
npm install vue-maplibre-kit vue maplibre-gl vue-maplibre-gl terra-draw @watergis/maplibre-gl-terradraw
```

## 导入样式

在应用入口或地图页面入口导入样式：

```ts
import "maplibre-gl/dist/maplibre-gl.css";
import "vue-maplibre-kit/style.css";
```

`vue-maplibre-kit/style.css` 是包的公开样式出口。外部项目不应该从构建产物或源码目录拼接样式路径。

## 接入原则

- 业务页面只依赖公开出口：`vue-maplibre-kit/business`、`vue-maplibre-kit/plugins`、`vue-maplibre-kit/config`、`vue-maplibre-kit/geometry`。
- 地图初始化参数仍然遵循 MapLibre GL 的 `MapOptions` 语义。
- 样式必须在地图组件渲染前导入，否则控件和弹窗可能出现布局异常。
