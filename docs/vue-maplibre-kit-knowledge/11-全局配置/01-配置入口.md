# 配置入口

公开入口：

```ts
import {
  defineMapGlobalConfig,
  setMapGlobalConfig,
  getMapGlobalConfig,
  resetMapGlobalConfig,
} from 'vue-maplibre-kit/config';
```

`src/config.ts` 只做薄转发，真实类型和运行时逻辑在 `src/entries/config.ts`。

## 推荐写法

`src/map-global-config.ts`

```ts
import { defineMapGlobalConfig, setMapGlobalConfig } from 'vue-maplibre-kit/config';

/**
 * 定义应用级地图默认配置。
 * @returns 地图全局默认配置
 */
export function createMapConfig() {
  return defineMapGlobalConfig({
    mapOptions: {
      mapStyle: 'https://demotiles.maplibre.org/style.json',
      center: [113.943, 22.548],
      zoom: 8,
    },
    mapControls: {
      MglNavigationControl: {
        isUse: true,
        position: 'top-left',
      },
    },
  });
}

/**
 * 在应用启动阶段注册地图默认配置。
 */
export function applyMapConfig(): void {
  setMapGlobalConfig(createMapConfig());
}
```

`src/main.ts`

```ts
import { createApp } from 'vue';
import App from './App.vue';
import { applyMapConfig } from './map-global-config';

applyMapConfig();

createApp(App).mount('#app');
```

## API 说明

| API | 作用 | 运行时副作用 |
| --- | --- | --- |
| `defineMapGlobalConfig(config)` | 提供类型辅助并原样返回配置对象 | 无 |
| `setMapGlobalConfig(config)` | 注册当前应用的全局配置快照 | 会深拷贝并冻结快照 |
| `getMapGlobalConfig()` | 读取当前全局配置快照 | 无 |
| `resetMapGlobalConfig()` | 清空当前全局配置 | 会回到空配置 |

## 冻结语义

`setMapGlobalConfig()` 会先深拷贝传入对象，再深度冻结。这样可以避免后续业务代码误改全局默认值，导致多个地图实例之间互相污染。

## 响应式对象提示

源码会用 Vue 的 `isReactive()` 和 `isRef()` 检查配置中是否包含响应式值。如果检测到，会输出提示：

```txt
[setMapGlobalConfig] 检测到 Vue 响应式配置值，建议传入普通对象快照
```

建议在注册前把响应式状态转换为普通对象，或者只把稳定常量写入全局配置。
