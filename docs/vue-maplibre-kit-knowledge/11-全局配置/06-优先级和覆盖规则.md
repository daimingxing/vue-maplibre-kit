# 优先级和覆盖规则

全局配置的定位是“应用级默认值”。页面配置和单次命令仍然可以覆盖它。

## 总体顺序

```txt
库内默认值
-> vue-maplibre-kit/config 全局默认值
-> 页面或实例配置
-> 单次命令 overrides
```

## mapOptions

`mapOptions` 使用顶层字段合并：

```txt
组件内置默认值 -> 全局 mapOptions -> 页面 mapOptions
```

对象型字段由后者整体接管，不做深度合并。

## mapControls

`mapControls` 按控件 key 合并，单个控件内部递归合并。页面局部字段优先，数组整体覆盖。

```txt
空对象 -> 全局 mapControls -> 页面 mapControls
```

## plugins

插件通常是：

```txt
插件内置默认值 -> 全局 plugins.* -> 当前插件实例配置 -> 单次命令 overrides
```

不同插件的合并粒度不同，应以各插件文档和源码为准。

## DXF 示例

DXF 导出有全局默认、插件实例默认和单次导出覆盖三层。

```ts
// 全局默认值
defineMapGlobalConfig({
  plugins: {
    dxfExport: {
      defaults: {
        sourceCrs: 'EPSG:4326',
        targetCrs: 'EPSG:3857',
        fileName: 'global-export.dxf',
      },
    },
  },
});

/**
 * 单次下载 DXF。
 * @param businessMap useBusinessMap 返回结果
 */
export async function downloadCurrentDxf(businessMap) {
  await businessMap.plugins.dxfExport.downloadDxf({
    fileName: 'current-page.dxf',
  });
}
```

最终文件名以单次 `overrides.fileName` 为准；未传的 `sourceCrs`、`targetCrs` 继续回落到更早层级。

## styles

图层样式工厂遵循：

```txt
库内样式默认值 -> 全局 styles.* -> 工厂调用参数或页面图层配置
```

`layout`、`paint` 内部字段会合并；表达式数组整体替换。

## 判断放在哪里

| 需求 | 推荐位置 |
| --- | --- |
| 所有页面统一底图、缩放范围、控件位置 | 全局配置 |
| 某个页面独有的图层、source、插件回调 | 页面实例配置 |
| 某次点击后的闪烁、显隐、临时 paint | `useBusinessMap` 命令 |
| 单次 DXF 文件名或过滤条件 | `downloadDxf(overrides)` / `exportDxf(overrides)` |
| 底层原生 API 才有的高级能力 | `rawHandles`，并记录清理逻辑 |
