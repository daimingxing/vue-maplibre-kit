# 运行时图层动作

适合谁读：需要临时添加 source/layer、修改图层显隐或写入 feature-state 的开发者。

先读哪篇：[04-业务数据源/03-MapBusinessSourceLayers](../04-业务数据源/03-MapBusinessSourceLayers.md)。

对应示例：[NGGI02](../../../examples/views/NG/GI/NGGI02.vue)、[NGGI03](../../../examples/views/NG/GI/NGGI03.vue)。

## 获取入口

```ts
const businessMap = useBusinessMap({
  mapRef: () => mapRef.value,
  sourceRegistry: registry,
});
```

## 添加临时 source/layer

```ts
const sourceResult = businessMap.layers.addGeoJsonSource("runtime-source", runtimeData);

if (sourceResult.success) {
  businessMap.layers.addLayer({
    id: "runtime-point-layer",
    type: "circle",
    source: "runtime-source",
    paint: {
      "circle-color": "#0ea5e9",
      "circle-radius": 10,
    },
  });
}
```

## 修改图层

| 方法 | 用途 |
| --- | --- |
| `hasSource` | 判断 source 是否存在 |
| `addGeoJsonSource` | 添加运行时 GeoJSON source |
| `removeSource` | 移除运行时 source |
| `hasLayer` | 判断图层是否存在 |
| `addLayer` | 添加运行时图层 |
| `removeLayer` | 移除运行时图层 |
| `setPaint` | 修改 paint 字段 |
| `setLayout` | 修改 layout 字段 |
| `setFeatureState` | 写入单要素 `feature-state` |

运行时动作适合临时分析、一次性高亮、调试图层；长期业务图层仍推荐声明式维护。
