# 自定义插件

当底层逻辑需要长期复用、需要渲染托管图层、需要贡献插件 API 或需要接入宿主交互时，应考虑自定义插件，而不是在页面里散落 `rawHandles` 逻辑。

## 入口

```ts
import { defineMapPlugin } from 'vue-maplibre-kit';
```

## 插件适合场景

- 需要在地图上长期维护一组运行时图层。
- 需要对外暴露插件状态和命令式 API。
- 需要接入普通图层交互或插件托管图层交互。
- 需要在销毁时统一清理事件、图层、source 和定时器。
- 需要多个业务页面复用同一套地图能力。

## 最小结构

```ts
import { defineMapPlugin } from 'vue-maplibre-kit';

/**
 * 创建示例地图插件。
 * @returns 地图插件定义
 */
export function createDemoPlugin() {
  return defineMapPlugin({
    type: 'demo-plugin',
    createInstance(context) {
      /**
       * 读取当前原生地图。
       * @returns 原生 MapLibre 地图实例
       */
      function getMap() {
        return context.getMap();
      }

      return {
        getApi() {
          return {
            /**
             * 输出当前地图是否就绪。
             * @returns 是否已取得地图实例
             */
            ping() {
              return Boolean(getMap());
            },
          };
        },
        destroy() {
          // 在这里清理插件创建的事件、图层、source 或定时器。
        },
      };
    },
  });
}
```

## 插件实例可提供的能力

- `getRenderItems()`：返回需要由宿主渲染的 Vue 组件。
- `getMapInteractivePatch()`：补充普通图层交互配置。
- `getPluginLayerInteractivePatch()`：补充插件托管图层交互配置。
- `resolveSelectedFeatureSnapshot()`：让宿主能从插件图层解析标准要素快照。
- `getApi()`：暴露命令式 API。
- `state`：暴露插件状态。
- `services`：贡献宿主服务，例如吸附或选择服务。
- `destroy()`：销毁插件资源。

## 与 rawHandles 的取舍

页面级一次性实验可以使用 `rawHandles`。一旦逻辑需要复用、需要清理、需要状态或需要参与宿主交互，就应转成插件。

## 边界

- 插件类型 `type` 在同一个地图实例内应保持唯一。
- 插件 ID 由插件描述对象管理，业务层通过宿主 `plugins.getApi(pluginId)` 或聚合门面读取。
- 插件不应直接修改其他插件维护的 source/layer，除非双方约定了公开 API。
- 自定义插件也应通过组件库公开入口暴露给外部项目消费，不应要求业务项目直接引用内部实现文件。
