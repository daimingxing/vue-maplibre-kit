# 何时使用逃生通道

逃生通道是高级能力，不是普通业务开发的首选路径。

## 首选：公开业务门面

这些场景优先使用 `vue-maplibre-kit/business`：

- 渲染业务 source 和图层。
- 读取选中态、保存属性、删除属性。
- 生成线延长草稿、替换线廊。
- 调用 snap、lineDraft、intersection、multiSelect、dxfExport。
- 写 feature-state、闪烁高亮、临时显隐图层。

## 其次：内置低层门面

当业务聚合门面不方便时，可以使用已公开的低层门面，例如：

- `useMapLayerActions`
- `useMapPopupState`
- `createMapBusinessSource`
- `createMapBusinessSourceRegistry`
- `defineMapPlugin`

这些能力仍属于公开 API，可以被外部项目稳定消费。

## 最后：rawHandles

只有下面这类情况才考虑 `rawHandles`：

- 需要调用 MapLibre 原生 API，且 `businessMap.layers` 没有覆盖。
- 需要绑定非常规地图事件或执行复杂 `queryRenderedFeatures()`。
- 需要读取 `vue-maplibre-gl` 宿主态，例如挂载、加载或包装对象状态。
- 需要继续下探 TerraDraw 实例，调用 `getSnapshot()`、`addFeatures()`、`setMode()` 等原生能力。

## 不建议使用逃生通道的情况

- 只是想添加普通业务图层。
- 只是想临时改 paint/layout。
- 只是想导出 DXF、刷新交点、清空线草稿。
- 只是想读取当前选中要素。

这些都已有更高层门面。

## 使用前检查

1. 是否已经确认公开入口没有合适能力。
2. 是否已经确认地图和控件生命周期。
3. 是否需要在组件卸载时解绑事件或清理图层。
4. 是否会与声明式图层、插件图层或业务 source 冲突。
5. 是否需要把这段逻辑沉淀为自定义插件。
