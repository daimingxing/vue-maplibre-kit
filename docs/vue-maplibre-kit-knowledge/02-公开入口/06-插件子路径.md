# 插件子路径

适合：需要单插件深度定制、直接拿插件常量或插件类型的高级开发者。

先读：[plugins 插件入口](03-plugins插件入口.md)。

对应示例：大多数业务示例只用 `vue-maplibre-kit/plugins`，`NGGI00` 会少量使用单插件常量。

## 可用子路径

| 子路径 | 适合场景 |
| --- | --- |
| `vue-maplibre-kit/plugins/map-feature-snap` | 吸附插件高级配置、规则类型、插件工厂 |
| `vue-maplibre-kit/plugins/line-draft-preview` | 线草稿插件 source/layer/generatedKind 常量、类型、插件工厂 |
| `vue-maplibre-kit/plugins/intersection-preview` | 交点插件 source/layer/generatedKind 常量、样式配置、插件工厂 |
| `vue-maplibre-kit/plugins/polygon-edge-preview` | 面边线临时预览、边线样式、source/layer/generatedKind 常量 |
| `vue-maplibre-kit/plugins/map-feature-multi-select` | 多选插件高级类型、插件工厂 |
| `vue-maplibre-kit/plugins/map-dxf-export` | DXF 导出任务类型、resolver 类型、插件工厂 |

## 普通业务页面不需要这样开始

普通业务页面推荐：

```ts
import { createBusinessPlugins } from "vue-maplibre-kit/plugins";
```

只有在你确实不想使用业务预设，或者要单独拼插件描述对象时，才使用子路径：

```ts
import { createMapDxfExportPlugin } from "vue-maplibre-kit/plugins/map-dxf-export";

const plugins = [
  createMapDxfExportPlugin({
    defaults: {
      fileName: "custom.dxf",
    },
  }),
];
```

## 读取能力仍推荐走业务门面

即使插件通过子路径注册，业务页面读取状态和动作仍优先走：

```ts
const businessMap = useBusinessMap({
  mapRef: () => mapRef.value,
  sourceRegistry: registry,
});
await businessMap.plugins.dxfExport.downloadDxf();
```

## generatedKind 常量

单插件子路径会导出本插件产生的 `generatedKind` 常量：

- `vue-maplibre-kit/plugins/line-draft-preview`：`LINE_DRAFT_PREVIEW_EXTENSION_KIND`、`LINE_DRAFT_PREVIEW_CORRIDOR_KIND`。
- `vue-maplibre-kit/plugins/intersection-preview`：`INTERSECTION_PREVIEW_KIND`、`INTERSECTION_MATERIALIZED_KIND`。
- `vue-maplibre-kit/plugins/polygon-edge-preview`：`POLYGON_EDGE_PREVIEW_KIND`。

snap、multiSelect、dxfExport 不产生业务生成要素，没有 `generatedKind` 常量。
