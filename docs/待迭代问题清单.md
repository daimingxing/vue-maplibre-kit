### 🟡 高影响问题（3个）

#### 问题 1: 插件配置样板过于冗长
**影响**: 每个插件配置需要 50-200 行，业务开发者需要理解大量内部细节

**当前写法** (80 行):
```typescript
const lineDraftPreviewPlugin = createLineDraftPreviewPlugin({
  enabled: true,
  styleOverrides: {
    line: {
      layout: {},
      paint: {
        "line-color": createFeatureStateExpression({
          hover: "white",
          default: "#fa8c16",
        }),
        "line-width": createFeatureStateExpression({
          hover: 6,
          default: 5,
        }),
        "line-dasharray": [2, 1.2],
      },
    },
    fill: {
      layout: {},
      paint: {
        "fill-color": "#fa8c16",
        "fill-opacity": 0.18,
        "fill-outline-color": "#ff7a00",
      },
    },
  },
  onHoverEnter: (context) => { /* ... */ },
  onHoverLeave: (context) => { /* ... */ },
  onClick: (context) => { /* ... */ },
  onDoubleClick: (context) => { /* ... */ },
  onContextMenu: (context) => { /* ... */ },
});
```

**理想写法** (15 行):
```typescript
const lineDraftPreviewPlugin = createLineDraftPreviewPlugin({
  theme: 'warning', // 自动应用橙色系样式
  interactions: {
    onClick: (context) => { /* 只写业务逻辑 */ },
  },
});
```

**改进建议**:
- 提供 3-5 个预设主题 (`primary`, `warning`, `danger`, `success`)
- 简化样式覆写 API，只暴露高频调整项 (color, width, opacity)
- 交互回调改为可选，提供默认行为

**预计工作量**: 8 小时

#### 问题 2: 图层配置重复度高
**影响**: 每个图层需要重复声明 `propertyPolicy`、`style`、`geometryTypes`

**当前写法** (每个图层 30 行):
```typescript
const primaryBusinessLayers = [
  createCircleBusinessLayer({
    layerId: LAYER_IDS.circle,
    propertyPolicy: {
      fixedKeys: ["mark"],
    },
    style: {
      layout: circleLayout,
      paint: circlePaint,
    },
    geometryTypes: ["Point"],
    where: { mark: "hole" },
  }),
  createCircleBusinessLayer({
    layerId: LAYER_IDS.circleDec,
    propertyPolicy: {
      fixedKeys: ["name", "mark"],
    },
    style: {
      layout: circleLayout,
      paint: circlePaint,
    },
    geometryTypes: ["Point"],
    where: { mark: "dec" },
  }),
  // ... 更多图层
];
```

**理想写法** (批量创建):
```typescript
const primaryBusinessLayers = createLayerGroup({
  source: SOURCE_IDS.primary,
  defaultStyle: circleLayerStyle,
  defaultPolicy: { fixedKeys: ["mark"] },
  layers: [
    { id: 'circle', where: { mark: "hole" } },
    { id: 'circleDec', where: { mark: "dec" }, policy: { fixedKeys: ["name", "mark"] } },
  ],
});
```

**改进建议**:
- 提供 `createLayerGroup` 批量工厂
- 支持图层继承机制，子图层只写差异
- 提供自动分层工具 (按属性字段自动拆分图层)

**预计工作量**: 8 小时

#### 问题 3: 样式表达式嵌套过深
**影响**: 业务开发者需要理解 `createFeatureStateExpression` + 属性 helper 的组合逻辑

**当前写法** (40 行):
```typescript
const { layout: lineLayout, paint: linePaint } = createLineLayerStyle({
  paint: {
    "line-color": createFeatureStateExpression({
      isFlashing: "#ffff00",
      states: {
        demoStyled: "#ec4899",
      },
      selected: "#f97316",
      hover: "#00ff00",
      order: ["isFlashing", "demoStyled", "selected", "hover"],
      default: matchFeatureProperty("id", { line_1: "#ff0000" }, "#0000ff"),
    }),
    "line-width": createFeatureStateExpression({
      states: { demoStyled: 8 },
      selected: 7,
      hover: 6,
      order: ["demoStyled", "selected", "hover"],
      default: 3,
    }),
  },
});
```

**理想写法** (8 行):
```typescript
const lineStyle = createLineLayerStyle({
  baseColor: '#0000ff',
  baseWidth: 3,
  states: {
    hover: { color: '#00ff00', width: 6 },
    selected: { color: '#f97316', width: 7 },
    flashing: { color: '#ffff00' },
  },
  propertyRules: {
    color: { line_1: '#ff0000' },
  },
});
```

**改进建议**:
- 提供状态样式预设对象，避免每个属性单独写表达式
- 简化属性映射语法，支持对象字面量
- 提供默认优先级，只在需要时覆写

**预计工作量**: 6 小时
